# Proposal: 答题结果页重构

## Why

当前答题结果页（`查看成绩`按钮对应的页面）存在以下问题：

1. **信息密度不足** — 认一认/说一说只显示得分，看不出题目本身是什么
2. **判断逻辑混淆** — 渐进评分（3/2/1/0）和对错评分（3/0）共用同一种展示方式
3. **选项题目无追溯** — 辨一辨/识图选一只显示 ❌ ✅，看不到学生选了什么、正确答案是什么
4. **缺少图片回顾** — 答题时图片一闪而过，结果页无法回看题目内容

## What Changes

### 结果卡布局（单人）

每个学生一张结果卡，卡片内按环节分块展示：


### 展示规则

| 环节 | 题目图片 | 学生答案 | 正答案 | 得分 |
|------|---------|---------|-------|------|
| **认一认** | ✅ 显示3张 | — | — | 显示每题得分（3/2/1/0） |
| **辨一辨** | ✅ 显示大图+全部小图 | 红色框标注 | 绿色框标注 | 显示每题得分 |
| **说一说** | ✅ 显示3张 | — | — | 显示每题得分（3/2/1/0） |
| **识图选一** | ✅ 显示2张 | 红色框标注 | 绿色框标注 | 显示每题得分 |

### 标记规则

- **认一认/说一说**：`●` 完全正确（3分）`⚠️` 部分正确（2/1分）`×` 错误（0分）
- **辨一辨/识图选一**：学生选的图片左上角标 🔴 ，正确答案左上角标 🟢（两个可能重叠）
- 图片下方显示具体分值数字

### 大图预览

- 点击卡片内任意图片，弹出全屏遮罩层显示大图
- 点击遮罩层任意位置或按 ESC 关闭
- 弹出层支持左右滑动浏览同一环节的全部图片

### 列表页改动

- 当前 `查看成绩` 列表改为：**姓名 + 各环节小计分** 的紧凑展示
- 点击姓名展开该学生的完整结果卡（折叠展开交互）
- 折叠状态下列表只占一行，展开后卡片展开

## Capabilities

### Modified Capabilities

### New Capabilities

- `result-card`: 单学生结果卡组件，支持4环节分块展示、图片点击放大

## Impact

**改动范围**：
- `index.html` — 结果展示页面重构（改 `viewResults` 相关 UI）
- `index.html` — 新增大图预览弹窗组件
- `index.html` — 结果数据需记录辨一辨/识图选一的学生选择项（需扩展 `results` 数据结构和 `saveResults` 调用时机）

**数据模型变化**：
`result` 对象需新增字段，记录辨一辨和识图选一每题学生选的图片ID：

```js
// 当前结构（需扩展）
{ studentName: "张三", scores: { r: [3,0,3], d: [3,0,3], e: [3,2,3], s: [3,0] }, ... }

// 新增字段（辨一辨/识图选一记录）
{ studentName: "张三", scores: {...}, 
  discrimination: [{ studentChoiceId, correctId, imageUrl }, ...],
  imageSelect: [{ studentChoiceId, correctId, imageUrl }, ...] }
```

**注意**：此项改动影响 `results` 的 IndexedDB 存储结构，历史数据（已有比赛记录）需兼容处理或清空重存。
